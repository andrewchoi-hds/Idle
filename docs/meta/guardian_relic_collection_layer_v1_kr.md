# 호법/법보 수집 레이어 초안 v1

기준 시점: 2026-04-29

## 1) 목적
- 현재 선협 성장 루프 위에 "수집형 메타 레이어"를 얹는다.
- 과금/가챠를 붙여도 곧바로 `파워 판매`처럼 보이지 않도록 안전한 collection lane을 먼저 정의한다.
- 전투/돌파/환생 재미를 해치지 않으면서 장기 수집 동기를 만든다.

## 2) 큰 방향
- 코어 성장:
  - 경지
  - 전투
  - 돌파/도겁
  - 환생
- 수집 레이어:
  - 호법(동행/수호령)
  - 법보(잔향/유물)
- 원칙:
  - 코어 진행 불가를 수집 레이어가 직접 해결하면 안 된다.
  - 수집 레이어는 "선택지 확장"과 "연출/컬렉션 만족"을 중심으로 간다.

## 3) 호법 시스템

### 역할
- 전투 보조 패시브
- 돌파/도겁 준비 보조
- 오프라인/자동화 품질 보조

### 슬롯 구조
- 기본 1슬롯
- 확장 2슬롯까지
- 동시 장착은 적지만, 보유/수집 풀은 넓게

### 효과 타입
1. 전투형
   - 전투 속도
   - 전투 승률 안정화
   - 전투 보상 보정
2. 돌파형
   - 돌파 성공률 미세 보정
   - 도겁 사망률 완화
   - 영약/부적 효율 보조
3. 운영형
   - 오프라인 효율
   - 자동돌파 차단 완화
   - 세션 리포트/로그 편의

### 가드레일
- 핵심 생존률을 호법 단일 파츠가 책임지지 않는다.
- "없으면 진행 불가"가 아니라 "있으면 선택지가 달라진다" 수준으로 설계한다.

## 4) 법보 시스템

### 역할
- 빌드 방향성을 바꾸는 유물형 수집
- 직접 스탯 폭증보다 규칙 변형에 가깝게 설계

### 예시
- 도겁 시 경상 실패 쪽 가중치 완화
- 자동돌파 pause threshold 유연화
- 전투 승리 시 특정 재화 수급 테마 보정
- 환생 직후 초반 가속 보정

### 레이어 설계
- 법보는 "장비"가 아니라 "메타 장착 옵션"으로 취급
- 장비 드롭과 섞지 않고, collection progression으로 분리

## 5) 진척 구조

### 무료 루프
- 메인 퀘스트
- 마일스톤
- 이벤트 교환
- 장기 누적 보상

### 유료 루프
- 직접 파워 판매보다:
  - 호법/법보 수집 속도
  - cosmetic variation
  - 중복 정리 속도
  쪽을 우선

### 중복 처리
- 중복은 파편/영혼결정 같은 전환 재화로 환산
- 중복이 꽝처럼 느껴지지 않게 누적 진척을 보장

## 6) 가챠가 붙는다면

### 추천 구조
- 호법/법보 통합 수집 배너
- 호법 중심 배너 + 법보 보조 배너
- 시즌 한정 cosmetic 배너

### 반드시 필요한 장치
- 천장
- 확률 공개
- 중복 전환 재화
- 무료 수급 경로
- 선택 상점 또는 교환 상점

### 비추천 구조
- 도겁 생존률 핵심 수치를 유료 전용으로 잠금
- 자동전투/자동돌파 사용 가능 여부를 가챠 파츠에 의존
- 초기 인간계 속도 문제를 가챠 파워로 덮는 구조

## 7) 수익화와의 연결

### 1단계
- 스킨/이펙트/테마
- 슬롯/로그/리포트 QoL
- 월간 패스

### 2단계
- 호법/법보 collection layer 공개
- 무료 수급 + 교환 상점부터

### 3단계
- 그 다음 가챠형 시스템 도입 검토
- 이때도 first spend는 power보다 collection/comfort 중심이 안전

## 8) 밸런스 체크포인트
1. 호법/법보가 없을 때도 인간계/신선계 progression이 목표 시간 안에 돌아가야 한다.
2. 호법/법보가 있을 때는 "덜 막힌다/더 편하다/빌드가 달라진다"가 핵심이어야 한다.
3. 도겁 고위험 구간 완화는 collection layer 보너스가 아니라 기본 설계/준비 콘텐츠로 먼저 해결해야 한다.
4. collection layer는 core power deficit의 처방전이 아니라 long-term engagement layer여야 한다.

## 9) 추천 구현 순서
1. 호법/법보 rarity/slot/effect taxonomy 표 작성
2. 무료 수급 루프 설계
3. 중복 전환 재화 설계
4. collection UI/장착 UX 설계
5. 그 뒤에야 가챠 여부 결정

## 10) 팀 결정 질문
1. 호법과 법보를 같은 수집 풀로 볼지, 분리할지
2. collection power를 전투형/돌파형/운영형 중 어디에 더 둘지
3. 무료 수급과 유료 수급의 비율을 어떻게 둘지
4. 첫 과금 상품을 collection과 연결할지, cosmetic/QoL만 둘지

## 11) 제안
- 지금 단계에서는:
  - collection layer taxonomy
  - 무료 수급 루프
  - duplicate conversion
  세 가지를 먼저 문서화한다.
- 가챠는 그 다음이다.
- 즉, `가챠 먼저`가 아니라 `collection layer 먼저`가 맞다.
